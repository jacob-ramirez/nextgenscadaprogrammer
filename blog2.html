Clear Ignition Internal Database Project Edits

Hello programmers, in this post I'm going to discuss a small bug in 7.9 of Ignition that I've stumbled across and got a work around for it. If you are still using 7.9 like I am, then this is a kind of maintenance you should be doing. The company I work at, we are still using 7.9 because we moved to Ignition 8 and found a major bug in it and decided it was not product ready.

When you are working on a project in the designer, you're making edits, saving changes, and publishing changes. All of these get a Edit#. Every edit number is stored in the internal database. After having a project for a year or so, this can get fairly large. In 7.9 it does not ever clear those out by default.

Now if you're working in Ignition 8 and later, you don't need to worry about this; it clears out these edits at some point. So there is a work around for this to clear out the edits in the internal database just for 7.9.

You need to go to the projects screen on the configure tab of the gateway. Hit the more button on the project you want to apply this to. Export published or staging version of the project. Now click the "Upload project from a *.proj backup fileâ€¦" link at the bottom.

After selecting the project export, select the overture checkbox and click apply. I have done this on a project that people were using and it just notified users that an update is available. Then when you go to the project details screen, there won't be any edits on that page, they will all be gone.

The steps are also presented here in the user manual: https://docs.inductiveautomation.com/display/DOC79/Project+Restore

I hope this post helps you implement a maintenance task if you ever run into any problems with the internal database. Clearing out the edits can decrease the size of the database alot which can increase its performance.

Until next time, continue coding.
